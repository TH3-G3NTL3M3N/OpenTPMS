ANT+ Tire Pressure Profile

The ANT+ Bicycle Tire Pressure device profile is a standardized protocol. Any ANT+ head unit that supports tire pressure will see our sensor natively — no app or data field needed.

FieldDataResolution
Tire PressureCurrent gauge pressure0.01 PSI
TemperatureSensor temperature0.01°C
Sensor LocationFront (0x01) / Rear (0x02)—
Battery StatusOK / Low / Critical—
Cumulative RevolutionsFrom accelerometerOptional

Note: ANT+ tire pressure profile requires an ANT+ adopter license (~$100/year for small volume). This grants access to the profile spec and device number allocation.

BLE GATT Custom Service

Custom GATT service for phones, Garmin CIQ, and Karoo. Uses 128-bit UUIDs to avoid conflicts.

Characteristic Properties Description
PressureRead, NotifyCurrent pressure in 0.01 PSI (uint16)
TemperatureRead, NotifyCurrent temp in 0.01°C (int16)
Battery LevelRead, NotifyStandard BLE Battery Service (uint8, 0-100%)
Sensor LocationRead, Write0x01=Front, 0x02=Rear. Stored in flash.
Alert ThresholdRead, WriteLow-pressure alert in 0.1 PSI (uint16). Stored in flash.
TX IntervalRead, WriteBroadcast interval in seconds (uint8, 1-10). Default 3.
Baseline P/TRead, WriteReference pressure + temp for compensation calc. Set at inflation.

v1 Configuration Interface: Web Bluetooth

How It Works

User opens a webpage (hosted on GitHub Pages or similar) in Chrome on their phone. The page uses the Web Bluetooth API to scan for and connect to the sensor. No app install needed.

The page shows: current pressure, temperature, battery level. Allows setting: sensor location (F/R), alert threshold, TX interval, baseline pressure.

Limitations

iOS Safari: No Web Bluetooth support. iOS users need the Bluefy browser app ($0) or WebBLE app as a workaround until the native iOS/Android app is built.

Chrome Android/Desktop: Full Web Bluetooth support. Works out of the box.
